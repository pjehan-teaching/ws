Server-Sent Events

Présentation

Les Server-Sent Events (SSE) permettent au serveur d'envoyer des données en continu vers le client via une connexion HTTP persistante.

Contrairement au long polling, la connexion reste ouverte indéfiniment et le client n'a pas besoin de renvoyer de requête.

Format du protocole SSE

Le serveur envoie des données au format texte simple, chaque événement étant séparé par une ligne vide :

HTTP/1.1 200 OK
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive

data: {"user": "Alice", "message": "Bonjour !"}

event: notification
data: {"type": "info", "text": "Nouvel utilisateur connecté"}
id: 42

: commentaire ignoré par le client
MDN : Server-sent events

Configuration du serveur SSE

import express, { Request, Response } from 'express';

const app = express();
const clients = new Set<Response>();

app.get('/events', (req: Request, res: Response) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');
  res.setHeader('Access-Control-Allow-Origin', '*');

  // Envoyer un événement initial
  res.write('data: {"type":"connected"}\n\n');

  clients.add(res);

  req.on('close', () => {
    clients.delete(res);
  });
});

Envoyer des événements depuis le serveur

function sendEvent(res: Response, data: unknown, event?: string): void {
  if (event) {
    res.write(`event: ${event}\n`);
  }
  res.write(`data: ${JSON.stringify(data)}\n\n`);
}

function broadcast(data: unknown, event?: string): void {
  clients.forEach(client => sendEvent(client, data, event));
}

// Exemple : envoyer l'heure serveur toutes les secondes
setInterval(() => {
  broadcast({ time: new Date().toISOString() }, 'tick');
}, 1000);

Utilisation de l'API EventSource côté client

const eventSource = new EventSource('/events');

// Écouter les événements par défaut (data sans event)
eventSource.onmessage = (event: MessageEvent) => {
  const data = JSON.parse(event.data);
  console.log('Message reçu :', data);
};

// Écouter un type d'événement spécifique
eventSource.addEventListener('tick', (event: MessageEvent) => {
  const { time } = JSON.parse(event.data);
  document.getElementById('clock')!.textContent = time;
});

// Gestion des erreurs
eventSource.onerror = () => {
  console.error('Connexion SSE perdue, reconnexion automatique...');
};
MDN : EventSource

Reconnexion automatique et ID d'événement

SSE gère nativement la reconnexion. Le client renvoie le header Last-Event-ID pour ne pas manquer d'événements :

app.get('/events', (req: Request, res: Response) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  const lastId = req.headers['last-event-id'];

  if (lastId) {
    // Renvoyer les événements manqués depuis lastId
    const missed = getMissedEvents(Number(lastId));
    missed.forEach(event => {
      res.write(`id: ${event.id}\n`);
      res.write(`data: ${JSON.stringify(event.data)}\n\n`);
    });
  }

  // Délai de reconnexion suggéré (ms)
  res.write('retry: 3000\n\n');
});

Avantages et limitations

Avantages Limitations
API native, simple d'usage Communication unidirectionnelle (serveur → client uniquement)
Reconnexion automatique intégrée Pas de support Internet Explorer
Support des événements nommés Limité à 6 connexions simultanées par navigateur (HTTP/1.1)
Transport HTTP standard (proxies, CDN) Texte uniquement (pas de binaire)
