Le temps réel sur le web

Qu'est-ce que le temps réel ?

Une application web temps réel est capable de transmettre des informations à ses utilisateurs instantanément, sans qu'ils aient besoin de recharger la page.

Le défi est de taille car le protocole HTTP est fondamentalement unidirectionnel : c'est toujours le client qui initie la communication.

Cas d'usage concrets

  • Messagerie instantanée : Slack, Discord, WhatsApp Web
  • Notifications : réseaux sociaux, e-commerce
  • Tableaux de bord : métriques système, analytics en direct
  • Jeux multijoueurs : synchronisation des positions
  • Collaboration : Google Docs, Figma, tableaux blancs
  • Trading : cours boursiers en temps réel

Le modèle HTTP classique

Le protocole HTTP repose sur un modèle requête / réponse :

  1. Le client envoie une requête au serveur
  2. Le serveur traite la requête et renvoie une réponse
  3. La connexion est fermée

Le serveur ne peut pas contacter le client de sa propre initiative : il doit attendre d'être sollicité.

Limitations de HTTP pour le temps réel

  • Le serveur ne peut pas pousser de données vers le client
  • Chaque requête implique des headers HTTP lourds
  • Établir une connexion TCP/TLS a un coût (latence)
  • Le modèle stateless complique la gestion d'état

Les techniques disponibles

Technique Direction Latence
Polling Client → Serveur Élevée

Server-Sent Events Serveur → Client Faible

Mise en place de l'environnement

Nous allons utiliser Node.js et TypeScript tout au long de ce cours.

Depuis Node.js 22.18, le TypeScript s'exécute nativement (sans ts-node) grâce au type stripping : vérifiez que votre version est à jour.

# Vérifier Node.js et npm (>= 22.18 requis)
node -v
npm -v

# Initialiser le projet et installer TypeScript localement
npm init -y
npm install express # HTTP framework
npm install --save-dev typescript @types/node @types/express

projet/

Mise en place de l'environnement

Nous allons maintenant modifier le fichier package.json :

{
    "scripts": {
        "watch:server": "node --watch --import=tsx src/server/server.ts",
        "watch:client": "tsc --watch",
        "typescript": "npx tsc --noEmit"
    },
    "type": "module"
}

Configuration TypeScript

// tsconfig.json
{
  "compilerOptions": {
    "rootDir": "src/",
    "outDir": "src/client/dist/",
    "target": "ES2020",
    "module": "esnext",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src/client/**/*.ts"]
}

Serveur HTTP de base

// src/server/server.ts
import express from 'express';
import path from 'path';

const app = express();

app.use(express.json());
app.use(express.static(path.join(import.meta.dirname, '../client')));

app.listen(3000);
