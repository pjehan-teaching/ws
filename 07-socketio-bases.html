Présentation de Socket.io

Socket.io est une bibliothèque qui abstrait la communication temps réel et ajoute de nombreuses fonctionnalités au-dessus des WebSockets natifs.

  • Reconnexion automatique
  • Fallback vers long polling si WebSocket indisponible
  • Événements personnalisés
  • Rooms et namespaces
  • Accusés de réception (acknowledgements)
Documentation officielle Socket.io v4

Socket.io n'est pas un WebSocket "pur"

Socket.io utilise son propre protocole applicatif au-dessus du transport (WebSocket ou HTTP) :

  • Un client Socket.io ne peut pas se connecter à un serveur WebSocket natif (et vice-versa)
  • Le premier échange se fait toujours en HTTP long polling, avant de basculer sur WebSocket
  • Ce comportement peut être forcé : transports: ['websocket']

Installation

# Côté serveur
npm install socket.io

# Côté client (si module bundler)
npm install socket.io-client

# Types TypeScript inclus dans le package socket.io

Socket.io v4 inclut les types TypeScript nativement, aucune installation de @types supplémentaire n'est nécessaire.

Configuration du serveur Socket.io

import express from 'express';
import { createServer } from 'http';
import { Server } from 'socket.io';

const app = express();
const httpServer = createServer(app);
const io = new Server(httpServer, {
  cors: {
    origin: 'http://localhost:3000',
    methods: ['GET', 'POST']
  }
});

io.on('connection', (socket) => {
  console.log(`Client connecté : ${socket.id}`);

  socket.on('disconnect', (reason) => {
    console.log(`Client déconnecté (${reason}) : ${socket.id}`);
  });
});

httpServer.listen(3000);

Événements personnalisés côté serveur

io.on('connection', (socket) => {

  // Écouter un événement du client
  socket.on('chat:message', (data: { author: string; text: string }) => {
    console.log(`Message de ${data.author} : ${data.text}`);

    // Diffuser à tous les clients (y compris l'émetteur)
    io.emit('chat:message', {
      ...data,
      id: Date.now(),
      timestamp: new Date().toISOString()
    });
  });

  socket.on('user:typing', (username: string) => {
    // Diffuser à tous sauf l'émetteur
    socket.broadcast.emit('user:typing', username);
  });

});

Configuration du client Socket.io

<script type="importmap">
    {
        "imports": {
            "socket.io-client": "https://cdn.jsdelivr.net/npm/socket.io-client@4.8.1/+esm"
        }
    }
</script>
import { io, Socket } from 'socket.io-client';

const socket: Socket = io('http://localhost:3000', {
  autoConnect: true,
  reconnection: true,
  reconnectionAttempts: 5,
  reconnectionDelay: 1000
});

socket.on('connect', () => {
  console.log(`Connecté avec l'id : ${socket.id}`);
});

socket.on('disconnect', (reason) => {
  console.log(`Déconnecté : ${reason}`);
});

socket.on('connect_error', (error) => {
  console.error('Erreur de connexion :', error.message);
});

Écouter et émettre des événements côté client

// Écouter un événement du serveur
socket.on('chat:message', (data: {
  id: number;
  author: string;
  text: string;
  timestamp: string;
}) => {
  displayMessage(data);
});

// Émettre un événement vers le serveur
function sendMessage(text: string, author: string): void {
  socket.emit('chat:message', { text, author });
}

// Indiquer que l'utilisateur est en train de taper
const input = document.getElementById('message-input')!;
input.addEventListener('input', () => {
  socket.emit('user:typing', 'Alice');
});

Les différents modes d'émission

// Envoyer au client actuel uniquement
socket.emit('event', data);

// Envoyer à tous les clients sauf l'émetteur
socket.broadcast.emit('event', data);

// Envoyer à tous les clients connectés
io.emit('event', data);

// Envoyer à une room spécifique
io.to('room-name').emit('event', data);

// Envoyer à tous dans la room sauf l'émetteur
socket.to('room-name').emit('event', data);

// Envoyer à un socket spécifique (message privé)
io.to(socketId).emit('event', data);

Accusés de réception (acknowledgements)

Socket.io permet de confirmer la réception d'un événement, comme un callback :

// Côté client : attendre une confirmation du serveur
socket.emit('chat:message', { text: 'Bonjour' }, (response: { ok: boolean; id: number }) => {
  if (response.ok) {
    console.log('Message enregistré avec l\'id :', response.id);
  }
});

// Côté serveur : envoyer la confirmation
socket.on('chat:message', (data, callback) => {
  const saved = saveMessage(data);
  callback({ ok: true, id: saved.id });
});
