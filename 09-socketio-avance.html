Socket.io : Namespaces

Créer des namespaces

import { Server } from 'socket.io';

const io = new Server(httpServer);

const chatNsp = io.of('/chat');
const notifNsp = io.of('/notifications');
const adminNsp = io.of('/admin');

chatNsp.on('connection', (socket) => {
  console.log('Connexion au namespace /chat :', socket.id);
});

notifNsp.on('connection', (socket) => {
  console.log('Connexion au namespace /notifications :', socket.id);
});

// Côté client
const chatSocket = io('http://localhost:3000/chat');
const notifSocket = io('http://localhost:3000/notifications');

Middleware d'authentification

Les middlewares Socket.io s'exécutent à chaque nouvelle connexion, avant l'événement connection :

import { Socket } from 'socket.io';

io.use((socket: Socket, next) => {
  const token = socket.handshake.auth.token as string | undefined;

  if (!token) {
    return next(new Error('Token manquant'));
  }

  try {
    const payload = verifyToken(token);
    socket.data.userId = payload.userId;
    socket.data.username = payload.username;
    next();
  } catch {
    next(new Error('Token invalide'));
  }
});

Envoyer le token depuis le client

const token = localStorage.getItem('token') ?? '';

const socket = io('http://localhost:3000', {
  auth: {
    token
  }
});

socket.on('connect_error', (error: Error) => {
  if (error.message === 'Token manquant') {
    // Rediriger vers la page de connexion
    window.location.href = '/login';
  }
});

Messages privés

Chaque socket possède son propre ID qui peut servir d'adresse pour les messages privés :

io.on('connection', (socket) => {

  socket.on('private:message', (data: {
    recipientId: string;
    text: string;
  }) => {
    const message = {
      from: socket.data.username,
      fromId: socket.id,
      text: data.text,
      timestamp: new Date().toISOString()
    };

    // Envoyer au destinataire
    io.to(data.recipientId).emit('private:message', message);

    // Confirmer à l'émetteur
    socket.emit('private:message:sent', {
      to: data.recipientId,
      ...message
    });
  });

});

Typage TypeScript des événements

Socket.io v4 permet de typer strictement tous les événements :

interface ServerToClientEvents {
  'chat:message': (data: Message) => void;
  'user:typing': (username: string) => void;
  'room:members': (data: { roomName: string; members: string[] }) => void;
  'private:message': (message: PrivateMessage) => void;
}

interface ClientToServerEvents {
  'chat:message': (data: { text: string }, callback: (res: { ok: boolean }) => void) => void;
  'room:join': (roomName: string) => void;
  'private:message': (data: { recipientId: string; text: string }) => void;
}

interface SocketData {
  userId: string;
  username: string;
}

const io = new Server<ClientToServerEvents, ServerToClientEvents, {}, SocketData>(httpServer);

Gestion des erreurs et déconnexions

io.on('connection', (socket) => {

  socket.on('disconnect', async (reason: string) => {
    // Récupérer toutes les rooms du socket avant sa déconnexion
    const rooms = [...socket.rooms].filter(r => r !== socket.id);

    for (const room of rooms) {
      socket.to(room).emit('room:user-left', {
        userId: socket.id,
        username: socket.data.username,
        roomName: room
      });
    }

    io.emit('user:disconnected', {
      username: socket.data.username,
      reason
    });
  });

  socket.on('error', (error: Error) => {
    console.error(`Erreur socket ${socket.id} :`, error.message);
  });

});

Scalabilité avec Redis Adapter

Pour déployer Socket.io sur plusieurs instances Node.js, il faut synchroniser les événements entre les serveurs :

npm install @socket.io/redis-adapter redis
import { createClient } from 'redis';
import { createAdapter } from '@socket.io/redis-adapter';

const pubClient = createClient({ url: 'redis://localhost:6379' });
const subClient = pubClient.duplicate();

await Promise.all([pubClient.connect(), subClient.connect()]);

io.adapter(createAdapter(pubClient, subClient));

Bonnes pratiques de sécurité

  • Toujours configurer CORS avec une liste blanche d'origines
  • Valider et assainir toutes les données reçues du client
  • Utiliser HTTPS / WSS en production (jamais ws:// en clair)
  • Authentifier chaque connexion via un token JWT court (expiration)
  • Limiter le débit des messages par client (rate limiting)
  • Ne jamais faire confiance à socket.id comme identifiant permanent
